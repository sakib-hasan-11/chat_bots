# AI Memory Chatbot with Hybrid Long-Term Memory

An end-to-end AI chatbot that combines **short-term conversational memory**, **structured long-term memory**, and **vector-based semantic retrieval** to simulate persistent user memory across conversations.

The project demonstrates a production-inspired memory architecture using **FastAPI**, **Redis**, **MySQL**, **OpenAI**, and **Pinecone**.

---

# Features

* Hybrid memory architecture
* Persistent user profiles
* Automatic fact extraction using LLM
* Semantic long-term memory retrieval
* Prototype-based retrieval classifier
* FastAPI backend
* Streamlit frontend
* Redis caching
* MySQL conversation storage
* Pinecone vector database
* OpenAI Embeddings

# Memory Design

The chatbot maintains three different memory layers.

## 1. Short-Term Memory

Stored in Redis.

Contains only the most recent conversation history for fast inference.

---

## 2. Persistent User Profile

Stored in Redis as a structured fact table.

Example:

```json
{
    "career_goals": [],
    "preferences": [],
    "skills": [],
    "projects": [],
    "current_focus": []
}
```

This profile is periodically regenerated by the LLM using the latest user conversations.

---

## 3. Long-Term Semantic Memory

Older fact tables are archived into Pinecone.

Each archived memory is:

* Converted into natural language
* Embedded using OpenAI Embeddings
* Stored in a user-specific namespace
* Retrieved through semantic similarity search

This enables the chatbot to remember information long after it has been removed from Redis.

# Key Design Decisions

* Redis is used for mutable working memory.
* MySQL stores complete conversation history.
* Pinecone stores archived semantic memories.
* Prototype embeddings reduce unnecessary vector searches.
* Structured fact tables minimize prompt size while preserving user context.
* User namespaces isolate memories across different users.
